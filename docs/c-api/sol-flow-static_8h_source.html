41 /* Static flow is a parent node that creates its children nodes and
42  * routes packets between them according to an specification received
43  * upon creation. The 'configuration' of the flow (connections made)
44  * will not change at runtime -- thus the name static).
45  *
46  * The specification consists in two arrays: one indicating each node
47  * type, and the other indicating each connection as a sequence of
48  * (source node, source port, destination node, destionation port)
49  * tuples.
50  *
51  * The connection specification is used to dispatch connections when
52  * messages are sent. This approach provides a very low memory
53  * overhead but still perfomant way to route the packets.
54  */
55
